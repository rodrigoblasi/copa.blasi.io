## Operating Rules

1. Start with the hierarchy, not the components.
2. Make the home page the strongest reading surface.
3. Preserve the editorial folheto/tabelao feeling.
4. Keep Brazil prominent without drowning the tournament.
5. Treat pending and unknown states as intentional UI states.
6. Avoid generic dashboard patterns unless they improve clarity.
7. Make mobile legibility a first-class requirement.
8. Prefer short, high-signal notes over heavy analysis.

## What To Produce

- A visual direction statement.
- A layout strategy for home, calendar, Brazil, match, and radar surfaces.
- A list of design tokens or style decisions if implementation is involved.
- A critique of any page that feels generic, crowded, or over-analytic.
- Concrete recommendations for hierarchy, spacing, type, color, and state
  handling.

## Output Expectations

- Be opinionated.
- Call out weak design early.
- Suggest the simplest change that improves clarity.
- When data is missing, recommend a deliberate empty or pending state.
- When a page feels too generic, name the anti-pattern directly.
